# Introduction to Instruction Level Parallelism

<TextWithImage
  body="Instruction-Level Parallelism (ILP) is one of the core concepts behind modern fast processors. 
        It is the architectural technique that allows a CPU to execute multiple instructions simultaneously, 
        ensuring that hardware resources are pushed to their maximum potential rather than sitting idle. 
        Every processor you interact with today, from the chip in a smartphone to a data-center server,

This exhibit explores **pipelining**, the primary hardware technique CPUs use to achieve ILP. 
We'll walk through why it was needed, how it works, what can go wrong when instructions overlap, and how 
modern hardware resolves those conflicts automatically.

## What is Pipelining?

Input Text Here

1. **Fetch (F):** Grab the instruction from memory.
2. **Decode (D):** Figure out what the instruction means and read the required registers.
3. **Execute (E):** Perform the mathematical or logical operation.
4. **Memory (M):** Read from or write to data memory.
5. **Write-back (WB):** Save the final result to a register.
